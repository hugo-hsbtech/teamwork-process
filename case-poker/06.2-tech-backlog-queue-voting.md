## Architecture Decisions (ADRs)

| # | Decision | Rationale |
|---|---|---|
| ADR-001 | Queue state stored as an ordered JSON array on the session record, not a separate table | Session-scoped, bounded (max 100 items). Avoids join overhead. Simpler migration — additive field on existing record. |
| ADR-002 | Vote values excluded from WebSocket broadcast payloads until `votes_revealed` event — enforced server-side | Client-trust is bypassable via DevTools. Server is the only safe enforcement point. Payload sends `{participant_id, status: "voted"}` only until reveal. |
| ADR-003 | `item_revealed`, `votes_revealed`, `item_skipped` added to existing event bus — no new broker | Low-frequency, session-scoped events. Existing infrastructure absorbs without scaling concern. |
| ADR-004 | Facilitator reconnect: session state snapshot on disconnect, restored within 5-min grace period | Prevents state loss without distributed lock. After 5 min, session closes cleanly with participant notification. |

## Definition of Done

A story is done when **all** of the following are true:

- [ ] All acceptance criteria defined in the Product Backlog story are met and verified by QA
- [ ] Vote concealment confirmed server-side — no vote value present in participant WebSocket payloads before `votes_revealed` (verified via network inspection)
- [ ] Unit tests cover the happy path and all edge cases defined in the story
- [ ] Code reviewed and approved by Tech Lead
- [ ] No regressions in existing session flow — QA regression suite passes
- [ ] Telemetry events confirmed received in staging pipeline
- [ ] Tech Lead closes the story in the backlog

## Rollout Strategy

1. **Feature flag** `feature_queue_voting` — disabled by default for all tenants on deploy
2. **Internal validation** — flag enabled on internal test account; PM + CS run a full ceremony simulation end-to-end
3. **Banco Meridional early access** — flag enabled for Banco Meridional 5 business days before general release; CS accompanies first live ceremony
4. **General availability** — flag enabled for all tenants after Banco Meridional confirms no issues
5. **Rollback** — disable `feature_queue_voting` flag; schema changes are additive and nullable — no rollback migration required
