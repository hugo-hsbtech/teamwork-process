## Architectural Decisions (ADRs)

| # | Decision | Justification |
|---|---|---|
| ADR-001 | Queue state stored as an ordered JSON array in the session record, not a separate table | Session-scoped, limited (max 100 items). Avoids join overhead. Simpler migration — additive field on the existing record. |
| ADR-002 | Vote values excluded from WebSocket broadcast payloads until the `votes_revealed` event — enforced server-side | Client trust is bypassable via DevTools. The server is the only secure enforcement point. Payload sends only `{participant_id, status: "voted"}` until reveal. |
| ADR-003 | `item_revealed`, `votes_revealed`, `item_skipped` added to the existing event bus — no new broker | Low-frequency, session-scoped events. Existing infrastructure absorbs them without scalability concerns. |
| ADR-004 | Facilitator reconnection: session state snapshot on disconnection, restored within the 5-min grace period | Prevents state loss without a distributed lock. After 5 min, session ends cleanly with participant notification. |

## Definition of Done (DoD)

A story is done when **all** of the following are true:

- [ ] All acceptance criteria defined in the Product Backlog story have been met and verified by QA
- [ ] Vote hiding confirmed server-side — no vote value present in participant WebSocket payloads before `votes_revealed` (verified via network inspection)
- [ ] Unit tests cover the happy path and all edge cases defined in the story
- [ ] Code reviewed and approved by the Tech Lead
- [ ] No regressions in the existing session flow — QA regression suite passed
- [ ] Telemetry events confirmed received in the staging pipeline
- [ ] Tech Lead closes the story in the backlog

## Rollout Strategy

1. **Feature flag** `feature_queue_voting` — disabled by default for all tenants on deploy
2. **Internal validation** — flag enabled on internal test account; PM + CS run a full end-to-end ceremony simulation
3. **Banco Meridional early access** — flag enabled for Banco Meridional 5 business days before general release; CS accompanies the first live ceremony
4. **General availability** — flag enabled for all tenants after Banco Meridional confirms no issues
5. **Rollback** — disable `feature_queue_voting` flag; schema changes are additive and nullable — no rollback migration needed
